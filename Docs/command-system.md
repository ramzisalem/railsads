🧠 Command System
The command system defines how users interact with AI efficiently.

Core principle
Guide user input instead of relying on open-ended prompts

Why this matters
Most AI tools fail because:
Users don’t know what to type

Goal
Make interaction fast, guided, and predictable

🎯 1. Command Types

1. Quick Actions (Primary)

Description
Predefined, clickable commands

Examples
Make it shorter
Make it more emotional
Try a UGC style
Focus on benefits
Add urgency

UI
Buttons below outputs
Inline in chat

Rule
Always visible, always relevant

2. Suggested Prompts

Description
Dynamic suggestions based on context

Example
Based on current creative:

→ Try a problem-focused angle
→ Target a different ICP
→ Make it more premium

Behavior
Changes after each generation
Context-aware

Rule
Suggestions should evolve with the user

3. Free Input (Chat)

Description
User writes custom instructions

Example
Make this more direct and aggressive
Rewrite for busy moms
Shorter headline

Rule
Free input = advanced layer, not primary

🧠 2. Command Hierarchy

Priority
1. Quick actions
2. Suggestions
3. Free input

Why
Reduce thinking

Guide behavior

🧠 6. Command Behavior Rules

Rule 1
Commands should feel like editing, not generating

Rule 2
Never overwhelm with too many options
(3–5 max)

Rule 3
Always provide next best action

Rule 4
Commands should be simple and actionable

⚠️ Anti-Patterns

❌ Empty input
No suggestions → user stuck

❌ Too many commands
Overwhelming UI

❌ Generic suggestions
Not tailored to context
